cendas.add_zero

cendas.state_column(data_frame: pandas.DataFrame(), code_column: str)

Parameters/Arguments

  • data_frame
    • A pandas dataframe.
  • code_column
    • The name of the column in the dataframe containing census tractcode/blockcode data as a string.

Description

Takes a pandas dataframe and the name of the dataframe column containing census tractcode/blockcode data as arguments. Parses tractcode/blockcode data to apply missing leading zeros to the codes if necessary. Returns a dataframe with the tractcodes/blockcodes correctly formatted.

The method assumes the following:

  • The user's tractcode or blockcode data is already converted to type string as opposed to type int or float.
  • The user's is missing leading zeros from their tractcode/blockcode data.

Why This Method is Useful

In many instances, tractcode and blockcode data are read as type int or type float as opposed to type string. When this happens, this causes the leading zeros to be dropped from the tractcode/blockcode data. Given the fact that the first 2 digits of every tractcode/blockcode indicate which state a tract or block is located in, dropping leading zeros will cause problems with analyzing codes starting with digits '01' all the way to '09.' For example, the tractcode '01001020100' is located in Alabama because the first 2 digits '01' correspond with the state of Alabama. However, if the leading zero was dropped because the tractcode data was read as type int or float as opposed to type string, we'd have '1001020100' as the tractcode instead, where the first 2 digits '10' correspond with the state of Delaware. The exclusion of leading zeros from tractcode/blockcode data can cause huge inaccuracies in geographic analyses. As such, it is necessary to verify that all the necessary leading zeros have been included in the data. The cendas.add_zero method allows a user to ensure that all the necessary leading zeros have been included within their tractcode/blockcode data.
